# SYSTEM OVERVIEW — AI_ASSISTANT_CORE

## Objectif du projet
Construire un orchestrateur IA local orienté création numérique, capable de :
- classifier une demande
- choisir un agent et un modèle
- décider d’un outil externe si nécessaire
- construire un plan d’exécution
- exécuter ce plan
- retourner une réponse finale propre avec surface de trace suffisante

Vision long terme :
→ copilote multi-outils local pour code, image, web et workflows créatifs.

## Topologie runtime canonique post-VM
### Dans la VM Hyper-V `AICORE-VM`
- backend AI_ASSISTANT_CORE
- `aicore-backend.service`
- SearXNG
- `aicore-searxng.service`

### Sur le host Windows
- Ollama
- ComfyUI
- éventuellement OpenWebUI comme UI opérateur distincte du runtime principal

### Frontière produit
- réseau privé Hyper-V `AICORE-INT`
- host : `192.168.77.1`
- VM : `192.168.77.10`
- le runtime principal vit dans la VM
- la VM fait partie du produit final et constitue une première couche de sécurité structurelle

### Bridge Ollama réel validé
- le chemin direct VM → `192.168.77.1:12000` ne fait pas partie du setup runtime validé
- le chemin actuellement validé est `VM -> http://192.168.77.1:12001`
- ce point d’entrée repose sur un `portproxy` host `192.168.77.1:12001 -> 127.0.0.1:12000`
- ce bridge est canonique à court terme et transitoire dans sa forme
