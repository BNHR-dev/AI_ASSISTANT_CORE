# API & INTEGRATION SURFACE — AI_ASSISTANT_CORE

## API FastAPI observée
### `GET /health`
Retourne :
- `status = ok`

### `GET /health/runtime`
Retourne :
- `status`
- `version`
- `checked_at`
- `summary`
- `services[ollama|searxng|comfyui]`

Cette route est désormais la surface canonique de validation du runtime post-VM.

### `GET /debug/canonical`
Retourne :
- `status`
- `version`
- `canonical_paths`
- `legacy_shims`
- `dormant_modules`
- `rule`

### `POST /route`
Entrée :
- `message`
- `has_image`

Retour :
- `task_type`
- `primary_agent`
- `selected_model`
- `needs_web`
- `second_call`
- `output_format`
- `selected_tool`
- `matched_rule`
- `reason_debug`
- `classifier_reason`
- `decision_trace`
- `decision_path`
- `reason`

### `POST /execute`
Entrée :
- `message`
- `has_image`

Retour :
- tous les champs de `/route`
- `execution_strategy`
- `execution_summary`
- `request_id`
- `started_at`
- `finished_at`
- `duration_ms`
- `plan`
- `step_results`
- `primary_output`
- `second_output`
- `artifact_type`
- `artifact_path`
- `artifact_filename`
- `artifact_paths`
- `artifact_filenames`
- `workflow_id`
- `comfyui_status`
- `comfyui_prompt_id`
- `variants_count`
- `completed_variants`
- `partial_visual_success`
- `output`

## Couche OpenAI-compatible
### `/v1/models`
Expose :
- `assistant-core-auto`
- `assistant-core-prof`
- `assistant-core-builder`
- `assistant-core-archi`
- `assistant-core-exam`

### `/v1/chat/completions`
Compatibilité minimale type Chat Completions pour OpenWebUI.
Prend en charge la détection de contenu image dans le dernier message utilisateur.

## Intégration OpenWebUI observée
- custom API OpenAI activée
- base locale typique : `http://host.docker.internal:8000/v1`
- recherche web activée avec moteur `searxng`
- URL SearXNG interne typique : `http://searxng:8080/search?q={query}&format=json`

## Lecture post-VM
- le backend Python principal vit désormais dans la VM
- SearXNG utilisé par le backend Python vit désormais dans la VM
- les backends host utiles sont joints via IP privée (`192.168.77.1`) depuis la VM
- dans l’état validé actuel, Ollama est joint via `http://192.168.77.1:12001` et non comme accès direct en `12000`
