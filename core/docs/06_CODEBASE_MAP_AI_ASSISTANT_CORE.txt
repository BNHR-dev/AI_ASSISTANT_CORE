# CODEBASE MAP — AI_ASSISTANT_CORE

## Noyau canonique

### Entrée API
- `app/main.py`
  → FastAPI ; endpoints `/health`, `/health/runtime`, `/debug/canonical`, `/route`, `/execute` ; inclut le routeur OpenAI-compatible

### Compréhension / décision
- `app/task_classifier.py`
  → classifier pondéré
- `app/tool_selector.py`
  → suggestion tool
- `app/engine/task_routing.py`
  → mapping statique agent / modèle / web / output_format
- `app/engine/routing_conditions.py`
  → règles hybrides contrôlées
- `app/engine/router_service.py`
  → fusion et décision finale

### Planification
- `app/engine/planner_types.py`
  → types de plan / état / résultat
- `app/engine/plan_builder.py`
  → convertit une décision en steps
- `app/engine/planner_service.py`
  → façade de build du plan
- `app/engine/state_store.py`
  → initialise l’état d’exécution

### Exécution
- `app/engine/step_executor.py`
  → exécution des steps
- `app/engine/result_assembler.py`
  → sortie finale visible
- `app/engine/executor.py`
  → orchestration d’exécution complète

### Prompts / contrats
- `app/engine/agent_prompt_registry.py`
  → prompts systèmes courts réellement injectés
- `app/engine/prompt_builder.py`
  → prompts d’étape
- `app/engine/output_contracts.py`
  → contrats de sortie par task_type
- `app/agents/prompts.py`
  → prompts historiques plus longs, non centraux au flux actuel

### Clients / tools
- `app/clients/ollama_client.py`
- `app/clients/web_client.py`
- `app/clients/comfyui_runtime.py`
- `app/clients/comfyui_client.py`
- `app/clients/blender_client.py`

### Visuel
- `app/engine/visual_types.py`
  → contrats `VisualRequest`, `VisualIntentAnalysis`, `VisualResult`
- `app/engine/visual_workflow_selector.py`
  → analyse d’intention visuelle + sélection de workflow
- `app/workflows/comfyui/*.json`
  → templates de workflow

### Runtime / observabilité
- `app/infra/tool_manager.py`
  → healthchecks runtime et autostart ComfyUI
- `app/engine/runtime_debug.py`
  → synthèse runtime + frontières canoniques

### Compatibilité / intégration UI
- `openai_compat.py`
  → expose `/v1/models` et `/v1/chat/completions`

## Legacy / compatibilité racine
- `executor.py`
- `router_service.py`
- `task_classifier.py`
- `tool_selector.py`
- `task_routing.py`
- `comfyui_client.py`

Rôle : shims de compatibilité, pas source métier.
