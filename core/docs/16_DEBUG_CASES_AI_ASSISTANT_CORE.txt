# DEBUG CASES — AI_ASSISTANT_CORE

## Cas 1 — Explain simple
Requête :
“explique moi les embeddings”

Attendu :
- `task_type = explain_basic`
- `strategy = single_step`

## Cas 2 — Explain + code
Requête :
“explique moi les embeddings avec un exemple python”

Attendu :
- `task_type = explain_basic`
- `second_call = build`
- `matched_rule = explain_plus_code`
- `strategy = two_step_llm`

## Cas 3 — Build simple
Requête :
“écris moi un script python simple”

Attendu :
- `task_type = build`
- `strategy = single_step`

## Cas 4 — Web research
Requête :
“cherche moi les dernières news IA”

Attendu :
- `task_type = web_research`
- `selected_tool = web`
- `strategy = web_pipeline`
- synthèse finale sans bruit technique intermédiaire

## Cas 5 — Image generation simple
Requête :
“génère une image cyberpunk”

Attendu :
- `task_type = image_generation`
- `selected_tool = comfyui`
- `strategy = visual_pipeline`
- si runtime indisponible : erreur propre et traçable

## Cas 6 — Intention visuelle portrait / poster
Requête :
“portrait pour une affiche de film cyberpunk”

Attendu :
- `subject_type = portrait`
- `render_intent = poster`
- `workflow_id = portrait_basic_v1`
- prompt enrichi avec signaux `poster composition`, `hero framing`, `cyberpunk aesthetic`

## Cas 7 — Intention visuelle produit / packshot
Requête :
“packshot de parfum luxe”

Attendu :
- `subject_type = product`
- `render_intent = packshot`
- `workflow_id = object_basic_v1`
- `style_flags` contient `luxury`
- format `1024 x 1024`
