# PROJECT SOURCES SLOT GUIDANCE — AI_ASSISTANT_CORE

## Objectif
Garder dans les sources projet uniquement les fichiers qui servent vraiment de mémoire fiable entre sessions.

## À garder en priorité
1. ce pack de contexte complet régénéré post-VM
2. `README.md` réécrit au niveau réel du projet
3. `ROADMAP.md` réécrite au niveau réel du projet
4. `ARCHITECTURE.md` réécrite au niveau réel du projet
5. le runbook post-VM
6. éventuellement une preuve de baseline stable si tu veux conserver un point de contrôle brut

## À ne pas utiliser comme source principale du projet ChatGPT
- les vieux README patchés
- les anciens récits de version trop minimale
- les fichiers racine legacy shim
- les caches / pyc / patch zips anciens
- les docs qui oublient la bascule post-VM du runtime

## Règle pratique
Dans le projet ChatGPT, mieux vaut :
- peu de sources mais canoniques
- des textes courts, denses, à jour
- un pack régénéré après chaque saut important

plutôt que :
- beaucoup de fichiers historiques contradictoires

## Moment où régénérer ce pack
- après changement de version déclarée
- après ajout d’endpoint de surface
- après évolution réelle du pipeline visuel
- après refonte des prompts / output contracts
- après nettoyage legacy ou debug surface significatif
- après tout saut réel de topologie runtime ou de frontière host / VM
